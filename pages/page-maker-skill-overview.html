page-maker

Un skill conçu pour transformer rapidement une discussion en cours en page HTML claire, stylée, lisible sur mobile et publiable. L’idée n’est pas de générer du HTML pour faire joli, mais de cristalliser une réflexion utile dans un format plus agréable à relire, partager et décider.

conversation → page
workspace + obe-pages
retourne le lien final
mobile-friendly

À quoi il sert

Le skill sert à prendre le signal d’une conversation et le figer dans une page lisible.

  • cadrage stratégique
  • comparatif d’options
  • architecture d’un système
  • plan d’action
  • récap / synthèse d’un échange
  • support de partage ou de revue plus tard

Quand l’utiliser

  • “fais-moi une page de ça”
  • “transforme cette réflexion en page”
  • “mets ça en page proprement”
  • “crée une page HTML sur ce sujet”
  • quand une conversation mérite d’être conservée dans un format plus clair

Workflow V1

1

Prendre la conversation utile

Le skill regarde le sujet en cours ou la portion récente de la discussion.

2

Extraire le signal

Objectif, options, décision, next steps, questions ouvertes.

3

Choisir la bonne forme

Architecture, comparatif, plan d’action, recap ou autre forme simple.

4

Générer la page HTML

Une page standalone, propre, stylée, pensée pour mobile.

5

Sauver et publier

La page va dans exports/, puis dans le repo obe-pages.

6

Retourner le lien final

Le skill doit finir en donnant le chemin local et l’URL GitHub Pages.

Ce que la V1 couvre

Oui

  • génération HTML stylée
  • sortie dans le workspace
  • publication dans obe-pages
  • retour du lien final
  • format pensé pour décision / lecture

Pas encore

  • template selector avancé
  • mise à jour automatique d’une page existante
  • index enrichi automatiquement
  • workflow interactif complexe

Pourquoi ce skill est intéressant

  • il transforme vite une pensée en artefact utile
  • il rend les synthèses plus agréables à consulter sur mobile
  • il crée une mémoire visuelle réutilisable
  • il réduit la friction entre conversation et documentation
  • il pousse vers plus de clarté sans créer de surprocess

Use cases typiques

  • expliquer une architecture produit
  • faire une page comparative de décisions
  • rendre un cadrage stratégique plus lisible
  • générer un recap partageable d’un sujet chaud
  • construire une base de pages consultables dans obe-pages
